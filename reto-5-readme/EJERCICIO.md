# Ejercicio: generador-readme

## Contexto de práctica

Este reto está diseñado para construir una skill de VS Code llamada **`generador-readme`**.
La skill debe generar un `README.md` útil y adaptado al tipo de proyecto.

## Entrada de prueba fija

- `reto-5-readme/detalles_proyecto.txt`

> No modifiques este archivo como parte del ejercicio. Es la entrada fija de evaluación.

## Qué debe hacer la skill

1. Leer notas desordenadas e informales.
2. Elegir uno de los tipos de proyecto descritos (API, librería o app).
3. Generar un README con secciones pertinentes para ese tipo.
4. Evitar usar siempre una plantilla rígida e idéntica.

## Casos borde obligatorios

- Proyecto API REST (ej. FastAPI): endpoints, ejecución, variables de entorno.
- Proyecto librería: instalación, uso, API pública, ejemplos.
- Proyecto app de escritorio: ejecución local, requisitos de plataforma, capturas o UX.
- Diferenciar claramente secciones esenciales según contexto.

## Entregable esperado de la skill

- `README.md` estructurado y contextual.
- Secciones que respondan al tipo de proyecto elegido.
- Contenido accionable (no relleno genérico).
